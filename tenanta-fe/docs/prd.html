Product Requirements (PRD)

Problem Statement

Landlords, managers, and tenants need a single platform to manage buildings, apartments, leases, payments, maintenance, and communication. Prospective tenants need a public listing and rental request flow.

Goals

Primary Goals
  • Enable admins to manage buildings and apartments.
  • Enable managers to operate within assigned buildings.
  • Enable tenants to view leases, payments, maintenance, chat.
  • Provide public listings and rental requests.
Success Metrics
  • 90%+ completion for building + apartment creation.
  • Key admin actions under 3 taps from dashboard.
  • Tenant onboarding under 2 minutes.

User Roles

Admin

Owner of buildings. Full management control.

Manager

Assigned to buildings by admin. Manages operations.

Tenant

Lives in apartment, views lease, payments, maintenance, chat.

Public

Unauthenticated visitors can view public apartments and submit rental requests.

Core Modules

  • Authentication and user profile
  • Building management
  • Apartment management
  • Tenant assignment and leases
  • Payments
  • Rental requests
  • Chat (conversations + messages)
  • Maintenance requests
  • Media uploads (buildings, apartments, maintenance, profile)
  • Dashboards (admin and tenant)

Business Rules

  • Only one active lease per apartment.
  • Apartment cannot be deleted if occupied.
  • Only owner can delete building.
  • Tenant can only view data for their lease/apartment.

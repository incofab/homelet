Overview

This documentation set defines the complete frontend specification for Tenanta (property management system) using Flutter + GetX. It covers the PRD, technical requirements, page inventory with interconnections, and the full API reference with request/response samples based on docs/api.md.

Platforms: Web (responsive) + iOS + Android
State Management: GetX (GetRouter + GetxController + reactive state)
API: REST (Laravel Sanctum Bearer tokens)
What’s Included
  • PRD and user roles
  • Flutter architecture and technical requirements
  • Page list with role-based access and navigation map
  • Routing, data flow, and sequence diagrams
  • Full API reference with sample requests and responses

Assumptions

  • API responses follow the envelope defined in docs/api.md.
  • Pagination is not currently defined; lists return full arrays.
  • Currency values are integer kobo and formatted as NGN in UI.
  • File uploads are multipart form-data with progress indicators.
  • Role-based UI is enforced server-side; client still handles 403.

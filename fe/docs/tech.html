Technical Requirements

Flutter
  • Flutter 3.x stable channel
  • Dart 3.x
  • Material 3 UI system
  • Adaptive layout for web and mobile
Core Packages
  • get for routing and state
  • dio for HTTP
  • flutter_secure_storage for token
  • image_picker or file_picker for uploads
  • intl for dates and currency
Environment
  • Dev, Staging, Prod base URLs in config.
  • Use flavors or compile-time defines.

State Management (GetX)

  • One GetxController per module.
  • Repositories encapsulate API calls and map to models.
  • Controller state with Rx types and immutable models.
  • Global session in AuthController with token + role.

Network Layer

  • Dio with interceptor for Bearer token.
  • Standard response decoding into ApiResponse<T>.
  • Centralized error handling for 401, 403, 422, and network failures.
